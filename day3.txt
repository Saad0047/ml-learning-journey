What I learned today
1) Decision Tree (Regression)
- A decision tree predicts a number (house price) by asking a chain of simple questions like:
  “Is area > 1500?”, “Is location good?”, etc.
- Each final group (leaf) predicts the average price of similar training examples.
- It can learn complex patterns, but a single tree can overfit (memorize training data).

2) Bagging vs Boosting (Simple idea)
- Bagging = “many independent opinions, then average them”
  This reduces random errors and makes predictions more stable.
- Boosting = “learn step-by-step from mistakes”
  Each new model focuses on fixing the errors made by the previous ones.

3) Random Forest Regressor (Bagging with Trees)
- Random Forest trains many decision trees and averages their predictions.
- This usually performs better than a single tree because averaging reduces overfitting.
- In my results, Random Forest (controlled) was the best tree-based model.

4) Gradient Boosting Regressor (Boosting with Trees)
- Gradient Boosting builds many small trees one after another.
- Each new tree tries to correct what the model is still getting wrong.
- It performed very close to Random Forest in my results.

Models trained and compared (with results)
- Random Forest (controlled) -> RMSE: 51240.62 | MAE: 34006.11
- Gradient Boosting (tuned) -> RMSE: 51799.53 | MAE: 35227.55
- Decision Tree -> RMSE: 67458.60 | MAE: 43036.13
- Poly(deg=2)+Ridge(alpha=10) -> RMSE: 65212.81 | MAE: 46612.52

Main conclusion from comparison
- Ensemble tree models (Random Forest / Gradient Boosting) beat the single Decision Tree
  and also outperformed Polynomial Regression + Ridge on this dataset.
- Best model today (tree algorithms): Random Forest (controlled)

Key takeaway (one line)
Ensemble methods combine many trees (by averaging or by step-by-step correction), and that usually gives better accuracy than a single model on complex data.
